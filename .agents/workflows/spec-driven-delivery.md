# Spec-Driven Delivery Workflow

## Purpose
Force every substantial task into a disciplined workflow.

## Required sequence
1. Read project constitution
2. Read relevant context files
3. Use skill-router
4. Restate the goal
5. Define constraints
6. Propose a plan
7. Create implementation checklist
8. Implement
9. Self-review against guardrails
10. Produce testing checklist
11. Summarize tradeoffs

## For coding tasks
Always output:
- Goal
- Files to create/edit
- Implementation plan
- Risks
- Validation checklist
- Final notes

## For design/UI tasks
Always output:
- Section/page goal
- Visual hierarchy
- Conversion role
- Premium risks
- Mobile implications

## For architecture tasks
Always output:
- Decision
- Why this is best now
- What is intentionally postponed
- Failure modes to avoid
