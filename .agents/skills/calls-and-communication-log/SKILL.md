# calls-and-communication-log

## Purpose
Shape the future communication layer around clarity and one client timeline.

## Use when
Later-phase call log and unified communication planning.

## Rules
- Keep a call list with answered/missed states.
- Allow private contacts to be hidden from business logs.
- Allow conversion from call to lead.
- Prefer one client timeline with filters.

## Guardrails
- Do not implement now unless task is explicitly about later app planning.
- Do not over-specify unsupported integrations.

## Output format
- Feature shape
- Data implications
- MVP / later split
