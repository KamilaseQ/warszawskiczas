# ui-system-guardrails

## Purpose
Protect the Warszawski Czas website from generic UI drift, visual clutter, fake-premium styling, and design inconsistency.

## Use when
Use for any public-facing UI decision in `apps/website`, especially when reviewing, refining, or approving layouts, sections, components, styling, and page-level polish.

## Do not use when
Do not use for backend-only work, admin tools, internal workflows, or tasks unrelated to public website UI.

## Project truths
- The website must feel premium, calm, selective, and trustworthy.
- The user wants strong design quality, not just functional UI.
- The visual system must stay elegant, readable, and future-proof.
- Accent colors are supportive details, not dominant design devices.

## Scope boundary
This skill is not responsible for:
- defining page intent
- defining page emotion
- designing section rhythm in detail
- full composition strategy

Those belong to:
- `wc-design-direction-and-page-intent`
- `wc-visual-grammar-and-composition`

This skill exists to stop visual mistakes and preserve design discipline.

## Core guardrail principle
Do not let the UI become:
- generic
- cluttered
- repetitive
- fake-premium
- trend-driven in a way that weakens timelessness

## Non-negotiable UI rules
- Keep hierarchy clear.
- Keep spacing generous enough to preserve class.
- Keep components clean and controlled.
- Keep accent usage restrained.
- Keep interactions elegant and readable.
- Keep the interface premium without becoming cold or empty.

## Forbidden patterns
- Generic ecommerce grid aesthetic
- SaaS-like section styling
- Card-heavy layout across the whole page
- Too many borders, dividers, frames, and decorative containers
- Heavy shadows used to fake depth
- Overuse of gold accents
- Overly dark luxury clichés
- Loud gradients
- Startup-style oversized rounded components
- Too many equally loud sections
- Elements that significantly reduce readability
- Visual decisions that make pages feel cheaper, busier, or more generic

## Component guardrails
- Reusable components must not flatten the site into one repetitive pattern.
- Components must support the brand, not dictate generic layouts.
- Buttons, cards, badges, and callouts must remain restrained.
- Avoid toy-like or trendy component styling.
- Avoid styling that feels too app-like for a luxury boutique website.

## Consistency guardrails
- Consistency does not mean every page looks identical.
- Shared UI patterns are allowed, but visual sameness is not the goal.
- Variation must stay inside one brand world.
- If a page feels disconnected, simplify and realign it.
- If a page feels too similar to every other page, do not force sameness just for consistency.

## Color guardrails
- Dark green and gold should be accents, not the foundation of every section.
- Gold should never become loud, glossy, or decorative noise.
- Do not use color to compensate for weak composition.
- Prefer restraint over visual signaling.

## Typography guardrails
- Headings must stay elegant and intentional.
- Body text must remain readable and calm.
- Avoid weak hierarchy.
- Avoid loud typographic tricks.
- Avoid long, bloated copy blocks when concise copy would be stronger.

## Mobile guardrails
- Mobile must remain premium, not just compressed desktop.
- Remove clutter aggressively.
- Protect hierarchy and CTA clarity.
- Avoid dense stacking of heavy elements.
- Maintain breathing room and calm.

## Review questions
- Does this feel more premium or just more decorated?
- Did this change improve clarity or only add styling?
- Is this UI becoming generic?
- Is this section/page too busy?
- Are accent colors doing too much work?
- Would simplification improve the result?

## Guardrails
- Do not replace composition with decoration.
- Do not replace restraint with emptiness.
- Do not replace elegance with trends.
- Do not accept UI that weakens prestige, readability, or scanability.

## Checklist
- Is the UI still calm and premium?
- Is the page free from generic patterns?
- Is the hierarchy still clear?
- Are accents restrained?
- Does mobile still feel elegant?
- Did we avoid visual clutter?

## Output format
- UI risks
- Guardrail violations
- What to simplify
- What to keep
