# owner-pwa-principles

## Purpose
Guide the future owner app toward a simple, installable, phone-first operational tool.

## Use when
Later-phase app planning, PWA decisions, owner workflow, mobile UX.

## Rules
- One-user-first design.
- Installable feel with home-screen icon.
- Fast, low-friction actions.
- Operational simplicity over feature richness.

## Guardrails
- Do not build this now unless the task is explicitly about the later app.
- Do not turn it into a full enterprise admin suite.

## Output format
- Owner workflow
- Mobile implications
- MVP / later split
