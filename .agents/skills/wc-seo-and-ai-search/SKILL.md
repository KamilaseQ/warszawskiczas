# wc-seo-and-ai-search

## Purpose
Build search visibility through high-quality content, technical cleanliness, strong commercial page structure, and clear local relevance.  
Use practical SEO ranges and rules of thumb, but never treat them as absolute ranking laws.

## Use when
Metadata, schema, page structure, search intent, local SEO, crawlability, indexability, internal linking, page quality, mobile usability, canonicalization, URL planning, content refreshes.

## Core principles
- People-first content first.
- Strong page purpose and commercial intent clarity.
- One strong page is better than several weak overlapping pages.
- Technical crawlability and mobile quality are mandatory.
- Use structured data only when truthful, visible, and useful.
- Internal linking should support discovery, hierarchy, and conversion.
- SEO decisions must support the business goal of the page.

## Practical SEO ranges and rules
These are operating ranges, not guarantees:

### Metadata
- Title tag target: **50–60 characters**
- Hard ceiling for title: **~65 characters**
- Every indexable page must have a **unique title**
- Primary topic/keyword should appear early in the title when natural
- Meta description target: **140–160 characters**
- Prefer **1 main keyword + 1 supporting variant** max in meta description
- Meta descriptions should be unique per important page
- Avoid uppercase-heavy titles and forced separators

### Headings
- Exactly **1 H1 per page**
- H1 should usually be very close to the title tag in meaning
- H1 target length: **under 70 characters**
- Use multiple H2s to structure the page logically
- H2/H3 should reflect subtopics, objections, use cases, or key decision points

### URLs
- Prefer short, descriptive, lowercase URLs
- Target URL length: **under 60 characters when possible**
- Avoid going beyond **100 characters** unless necessary
- Use hyphens, never underscores
- Remove unnecessary parameters from indexable URLs
- Keep one canonical version of each indexable page

### Content structure
- Primary topic should appear in:
  - title
  - H1
  - intro / first paragraph
  - naturally across the body
- Mention the primary query/topic in the **first 100–150 words** when it makes sense
- No fixed word-count rule; length should match search intent
- For competitive commercial or editorial pages, use top-ranking pages as a benchmark, not a law
- If benchmarking content length, compare against the **top 3–5 realistic competitors**
- Keyword density is not a target KPI; if checked at all, keep exact-match usage generally **below ~1–2%** and prioritize natural language

### Internal linking
- Every important SEO page should receive internal links
- Prefer descriptive anchor text, not “kliknij tutaj” / “read more”
- Important pages should ideally be reachable within **3 clicks** from the homepage or key hub pages
- Link from:
  - hub/category pages
  - related editorial pages
  - nearby commercial pages
- Reuse consistent anchor themes, but do not over-optimize with identical anchors everywhere

### Images
- Use descriptive filenames before upload
- Every meaningful image should have useful alt text
- Alt text should describe the image first, not stuff keywords
- Compress images before upload
- Serve responsive images
- Decorative images should not carry fake SEO alt text

### Technical SEO
- Site must run on **HTTPS**
- Key pages should return **200**
- Fix or redirect broken important URLs
- Use 301 redirects for replaced URLs
- Avoid redirect chains; keep them to **0–1 hop**, absolute maximum **2**
- Use canonical tags on indexable pages where duplication risk exists
- XML sitemap should include important canonical URLs only
- robots.txt must not block critical CSS/JS needed for rendering
- Block or noindex low-value pages when appropriate:
  - thank-you pages
  - cart
  - checkout
  - login/account
  - internal search results
- Ensure only pages meant to rank are indexable

### Mobile and UX
- Responsive design is mandatory
- Aim for page load of **under 3 seconds**
- Best-case aspiration for critical pages: **around 2 seconds**
- Base body font should generally be **16px**
- Mobile tap targets should be about **48x48 px**
- Spacing between tap targets: at least **8 px**
- Avoid intrusive elements that block content or interaction

### Local SEO
- Keep NAP (name, address, phone) fully consistent across the site and profiles
- Strong local landing pages should clearly state:
  - city/area served
  - trust signals
  - contact method
  - why choose this business locally
- Use Google Business Profile truthfully and keep hours, address, phone, and categories current
- Reviews matter for trust and local conversion, but never fake or mass-template them

### Schema
- Only add schema that matches visible page content
- Recommended when truthful:
  - Organization / LocalBusiness
  - Product
  - BreadcrumbList
  - FAQPage only for real on-page FAQs
- Never use fake FAQ/schema just to chase SERP features
- Schema is a support layer, not a substitute for page quality

## Page evaluation framework
For each page, output:
- Search intent
- Page role
- Primary conversion goal
- Metadata direction
- Heading structure direction
- Internal linking notes
- Schema notes
- Technical notes
- Risks

## Guardrails
- No keyword stuffing
- No fake FAQ/schema
- No filler pages
- No doorway/location-spam pages
- No writing for bots instead of users
- No invented SEO “facts” presented as Google rules
- No recommending meta keywords
- No forcing word count for its own sake
- No duplicating page purpose across multiple weak URLs

## Decision rules
- If two pages target the same intent, prefer merging or differentiating clearly
- If a page has weak unique value, strengthen it or remove it
- If content is thin but the intent is narrow, do not pad it artificially
- If a page is important commercially, make conversion intent obvious
- If structured data is not fully supported by visible content, do not add it
- If SEO best practice conflicts with user trust or brand quality, choose trust and clarity first
